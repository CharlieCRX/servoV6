# 急停后位置显示与编排器状态恢复 -- 问题分析与重构

> **文档版本**: v1 (2026-05-18)
>
> **相关日志**: 启动急停案例日志 -- 14:32:08 ~ 14:32:18

---

## 1. 问题诊断

### 1.1 Bug #1：急停触发后位置显示为零

**现象（日志摘录）**:
```
[14:32:12.750] Machine_A Y: pos=+42.0 MovingRelative   ← 运动中途，位置正确
[14:32:13.750] Machine_A Y: pos=+0.0  Unknown  errs=81  ← 急停后位置归零！状态变 Unknown
[14:32:14.750] Machine_A Y: pos=+0.0  Unknown  errs=381 ← 解除急停，位置仍为零
[14:32:15.751] Machine_A Y: pos=+80.5 Disabled errs=684 ← 解除后恢复显示正确位置
```

**根因分析**:

```
用户点击急停按钮
  -> EmergencyStopController::requestEmergencyStop()
    -> m_state = EmergencyStopping
    -> isSystemLocked() 返回 true
       ↓
AxisViewModelCore::absPos() 调用 tryGetAxis()
  -> SystemContext::tryGetAxis()
    -> Layer 0: isSystemLocked() == true -> return false (ContextRejection::SystemSafetyLocked)
    -> ⚠️ absPos() 返回 fallback 值 0.0
    -> ⚠️ state() 返回 fallback 值 AxisState::Unknown
```

**关键代码路径**:

```cpp
// AxisViewModelCore.cpp -- 状态投影
double AxisViewModelCore::absPos() const {
    auto* axis = tryGetAxis(m_manager, m_groupName, m_axisId);
    if (!axis) return 0.0;  // ← 急停期间永远走到这里
    return axis->currentAbsolutePosition();
}

AxisState AxisViewModelCore::state() const {
    auto* axis = tryGetAxis(m_manager, m_groupName, m_axisId);
    if (!axis) return AxisState::Unknown;  // ← 急停期间永远走到这里
    return axis->state();
}

// SystemContext.h -- tryGetAxis 安全联锁
bool tryGetAxis(AxisId id, Axis*& outAxis, ContextRejection& reason) {
    // Layer 0：安全域最高优先级拦截
    if (m_emergencyStopController.isSystemLocked()) {
        reason = ContextRejection::SystemSafetyLocked;
        outAxis = nullptr;
        return false;  // ← 阻断所有轴访问，包括只读遥测
    }
    // ...
}
```

**设计问题**: `tryGetAxis()` 将**控制操作拦截**和**遥测读取**混在一起。急停期间物理位置依然有效，但 UI 无法读取。

### 1.2 Bug #2：解除急停后编排器无法恢复，界面锁死

**现象（日志摘录）**:
```
[14:32:11.735] [Machine_A][Y] START MoveRelative distance=100
[14:32:12.750] Y: pos=+42.0 MovingRelative   ← 运动中
qml: EmergencyStopButton: 触发急停
[14:32:13.750] Y: pos=+0.0 Unknown errs=81   ← 急停阻断
qml: EmergencyStopButton: 解除急停
[14:32:15.751] Y: pos=+80.5 Disabled errs=684 ← 轴掉电，编排器依然 Error
[14:32:16.750] Y: pos=+80.5 Disabled errs=984 ← 错误持续累加
[14:32:17.750] Y: pos=+80.5 Disabled errs=1284
-> 界面锁死，无法发起新的位置移动
```

**根因分析**:

```
AutoRelMoveOrchestrator::tick() 调用链:
  -> group->tryGetAxis(m_targetId, axis, ctxReason)
    -> isSystemLocked() == true
    -> return false (ContextRejection::SystemSafetyLocked)
  -> m_step = Step::Error          ← ⚠️ 终端状态，永不恢复
  -> m_lastError = ctxReason      ← ⚠️ 每帧重复设错，errs 持续累加

急停解除后:
  -> isSystemLocked() 回到 false
  -> tick() 再次执行
  -> tryGetAxis() 成功
  -> switch(m_step): case Step::Error -> break  ← ⚠️ 什么也不做！
  -> 编排器卡死在 Error 态，无法发起新运动
```

**三个编排器都受影响**:
- `AutoRelMoveOrchestrator` -- Step::Error 是 sink state
- `AutoAbsMoveOrchestrator` -- Step::Error 是 sink state  
- `JogOrchestrator` -- Step::Error 是 sink state

**设计问题**: 编排器将 `SystemSafetyLocked` 视为不可恢复的"硬件故障"，但实际上急停是可逆操作。急停解除后，编排器应自动复位到 Idle/Initial 状态。

---

## 2. 重构方案

### 2.1 修复 Bug #1：分离遥测读取与控制访问

**策略**: 在 `SystemContext` 中新增 `tryReadAxis()` -- 仅绕过 Layer 0 安全联锁，仍保留龙门语义拦截（因为龙门解耦时 X 逻辑轴确实不存在）。

```
                  tryGetAxis()                     tryReadAxis()
               ┌────────────────┐              ┌──────────────────┐
  Layer 0      │ ✅ 安全锁定拦截 │              │ ❌ 不拦截         │
  Layer 1      │ ✅ 龙门未同步   │              │ ✅ 龙门未同步      │
  Layer 2      │ ✅ 龙门语义     │              │ ✅ 龙门语义        │
  Layer 3      │ ✅ 容器查找     │              │ ✅ 容器查找        │
               └────────────────┘              └──────────────────┘
  用途          控制指令（使能/运动）             遥测读取（位置/状态）
```

**变更文件**:
- `domain/entity/SystemContext.h` -- 新增 `tryReadAxis()`
- `presentation/viewmodel/AxisViewModelCore.cpp` -- `absPos()` / `relPos()` / `state()` 改用 `tryReadAxis()`

### 2.2 修复 Bug #2：编排器安全锁定降级处理

**策略**: 当编排器检测到 `SystemSafetyLocked`，不等同于硬件故障（Error），而是优雅退出到 Idle/Initial，允许后续重新发起操作。

新增检测逻辑：
- `tryGetAxis` 返回 `SystemSafetyLocked` -> 转入 `Step::Initial`（清理所有内部标志位）
- 运动中轴直接变为 `Disabled` -> 同样视为外部中断，优雅退出

**变更文件**:
- `application/policy/AutoRelMoveOrchestrator.h` -- 新增 SafetyLocked / DisabledDuringMotion 降级逻辑
- `application/policy/AutoAbsMoveOrchestrator.h` -- 同上
- `application/policy/JogOrchestrator.h` -- 同上
